# NarratorDB — Типы данных

## Value

Каждое значение кодируется как `[TYPE: 1 byte][DATA: N bytes]`.

| Байт   | Вариант           | Данные                                  | Описание                              |
|--------|-------------------|-----------------------------------------|---------------------------------------|
| `0x00` | `Null`            | 0 байт                                  | Не задано — fallback по цепочке LINK  |
| `0x01` | `Bool(bool)`      | 1 байт (`0x00` / `0x01`)               |                                       |
| `0x02` | `Int(i64)`        | 8 байт LE                               |                                       |
| `0x03` | `Float(f64)`      | 8 байт LE                               |                                       |
| `0x04` | `Text(String)`    | 4 байта длина + UTF-8                   |                                       |
| `0x05` | `Link(Link)`      | 4 байта длина + path + MAP (local)      | Ссылка с опциональными оверрайдами    |
| `0x06` | `Blob(Vec<u8>)`   | 4 байта длина + raw bytes               |                                       |
| `0x07` | `Array(Vec<Value>)`| 4 байта count + [value...]             |                                       |
| `0x08` | `Map(HashMap)`    | 4 байта count + [TEXT key + value...]   | Вложенный объект                      |
| `0x09` | `Date(u32)`       | 4 байта (дней с эпохи)                  |                                       |
| `0x0A` | `Time(u32)`       | 4 байта (секунд с полуночи)             |                                       |
| `0x0B` | `DateTime(u64)`   | 8 байт (unix timestamp)                 |                                       |
| `0x0C` | `Tombstone`       | 0 байт                                  | Явное отсутствие — цепочка стопорится |

---

## Null vs Tombstone

| Состояние              | Смысл                                              |
|------------------------|----------------------------------------------------|
| `Null`                 | Не задано здесь — смотреть выше по LINK-цепочке   |
| `Tombstone`            | Явно пусто — цепочка останавливается               |
| Ключ отсутствует в MAP | То же что `Null`                                   |

Пример:
```
john.body.left_arm  = Tombstone  → руки нет, поиск вверх не идёт
john.body.right_arm = Null       → смотрим тело расы через LINK
```

## Link

```rust
pub struct Link {
    pub path:  String,   // путь до целевой записи
    pub local: Map,      // локальные оверрайды (может быть пустым)
}
```

### Формат пути

```
collection/key[/subpath...]
```

Сегмент `key` может быть:
- Литералом: `items/item12321`
- Само-ссылкой: `races/$self.identity.race_id/body`

`$self.field.subfield` вычисляется из **корневой записи** (той, с которой начался резолв).

### Поведение

| `local` | Поведение |
|---------|-----------|
| Пустой  | Значение целиком берётся из целевой записи (простая ссылка) |
| Непустой | Ключи из `local` имеют приоритет, остальное берётся из целевой записи (прототипное наследование) |

### Примеры

```python
# Простая ссылка на запись
Link("items/item12321")

# Прототипное наследование — тело персонажа наследует от расы
Link("races/$self.identity.race_id/body", local={
    "left_arm":  TOMBSTONE,              # руки нет
    "right_leg": Link("items/item12321") # нога заменена протезом
})
```

## Map

`Map = HashMap<String, Value>` — вложенный объект без схемы.
Ключи — произвольные строки, значения — любые `Value`.

Используется как для хранения данных записи, так и внутри `Link.local`.
